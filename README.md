# keiba-research — verification-asset hash registry

日本中央競馬（JRA）の市場効率性に関する独立研究プログラムの、検証資産の
**sha256ハッシュのみ**を時系列で公開するレジストリです。データ・分析コード・
論文本文はここには置きません。

## このリポジトリの目的

1. **公開タイムスタンプ（優先権）の確保** — 各研究章の成果が確定した時点の
   ハッシュを、内容の公開に先立って記録します。
2. **「後出し不能」の第三者検証可能性** — 本研究は、結果を見る前に判定基準を
   固定する事前登録方式で運用しています。事前登録文書のハッシュが結果確定前に
   記録されていることを、誰でも git のコミット履歴から確認できます。

## 研究プログラムの概要

- **対象**: JRA公式の公開情報（期間: 2020〜2026年。主要レースを中心に構成した研究用コーパスで、
  8,951レース／出走馬単位127,662件。train/validation/blind/lockbox の
  4分割で、検証期間を時系列で厳密に分離）を対象とした、ベッティング市場の
  価格形成・効率性の実証分析。
- **中心的な問い**: 公開情報から構成できる予測シグナルのうち、市場オッズに
  織り込まれていない（＝統計的に検出可能な非効率が残っている）ものは
  存在するか。
- **規律**:
  - **事前登録** — 各章の分析開始前に、仮説・判定基準・パラメータを
    `preregistration.md` として凍結し、以後変更しない。
  - **独立検証期間での再現要求** — 所見は2つの独立した検証期間の両方で
    再現された場合のみ採用する。
  - **凍結資産** — 実行メタデータ（`run_meta.json`）・集計結果CSV・
    章本文は確定後に凍結し、上書きしない。
  - **時系列4分割** — train（2020-2023）/ validation（2024）/ blind（2025-2026前半）
    / lockbox（2026後半）に時系列で分割し、lockbox は最終検証まで一切参照しない。
- **これまでの主要所見**: 検討したシグナルの大部分について、市場オッズは
  公開情報を高い精度で織り込んでおり、執行可能な収益機会は検出されて
  いません（＝市場効率性を支持するヌル結果が主体です）。個別章の所見は
  連載記事およびプレプリントで順次公開します。

## 記録対象と照合方法

`HASHES.md` に記録するのは各章の凍結資産です:

| 資産 | 内容 |
|---|---|
| `preregistration.md` | 分析前に固定した仮説・判定基準 |
| `run_meta.json` | 実行時のパラメータ・乱数シード等のメタデータ |
| 凍結CSV | 確定した集計結果 |
| `chapter_text.md` | 章の本文（分析報告） |
| `paper*_draft_v*.md` | 論文の内部草稿 |

原本が公開された時点で（プレプリント公開時等）、第三者は次の1行で
照合できます:

```bash
sha256sum <入手した原本ファイル>
# 出力が HASHES.md の該当行と一致すれば、そのファイルは
# 記録日時の時点で確定していたことが検証できます
```

## 運用ルール

- `HASHES.md` は **append-only** です。行の削除・書き換えは行いません。
  ファイルが更新された場合は `updated` 注記付きの行を**追加**します。
- ハッシュの追記とコミットは研究パイプラインが章の確定時に自動で行い、
  本リポジトリへの push は人間が行います。
