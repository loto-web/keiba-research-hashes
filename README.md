# keiba-research — verification-asset hash registry

日本中央競馬（JRA）の市場効率性に関する独立研究プログラムの、検証資産の
**sha256ハッシュのみ**を時系列で公開するレジストリです。データ・分析コード・
論文本文はここには置きません。

## このリポジトリの目的

1. **公開タイムスタンプ（優先権）の確保** — 各研究章の成果が確定した時点の
   ハッシュを、内容の公開に先立って記録します。
2. **「後出し不能」の第三者検証可能性** — 本研究は、結果を見る前に判定基準を
   固定する事前登録方式で運用しています。原本が公開された時点で、第三者は
   ハッシュを照合することで「そのファイルが当該コミットの時点で確定していた」
   ことを検証できます。

**本レジストリが証明するのは、コミットの時点における各ファイルの同一性のみです。**
`HASHES.md` の `recorded=` 欄は研究側の自己申告であり、外部証跡はコミット履歴です。
外部証跡が及ぶ範囲は **2026-07-13 以降**に限られます（「既知の限界」参照）。

## 研究プログラムの概要

- **対象**: JRA公式の公開情報（2020〜2026年）。研究用コーパスは
  **各開催日の第8〜12レースに限定**しており、8,951レース／出走馬単位127,662件。
  これは JRA中央フラットレース全体の約38%（障害を含む JRA 市場全体では約21%）に
  相当します。
  **この限定は市場の性質によるものではなく、予測モデルが前走情報を必要としたことに
  由来する分析側の都合です。** したがって本プログラムの所見はいずれも
  「この観測窓の内側での所見」であり、新馬戦・未勝利戦・第1〜7レースを含む
  JRA市場全体についての所見ではありません。
- **中心的な問い**: 公開情報から構成できる予測シグナルのうち、市場オッズに
  織り込まれていない（＝統計的に検出可能な非効率が残っている）ものは存在するか。
- **時系列分割**（現行の正規定義）:

  | split | 期間 | 状態 |
  |---|---|---|
  | train | 2020-01-05 〜 2023-12-28 | 使用済 |
  | validation | 2024-01-06 〜 2024-12-28 | 使用済 |
  | blind | 2025-01-05 〜 2026-02-08 | **研究期間中に複数回測定済**（下記4） |
  | lockbox | 2026-02-09 〜（開区間・週次で伸長） | 2026-06-21 まで消費済／2026-06-22 以降は未参照 |

- **規律**:
  - **事前登録** — 各分析の計算実行前に、仮説・判定基準・パラメータを凍結し、以後変更しない。
  - **独立検証期間での再現要求** — 所見は独立した検証期間で再現された場合のみ採用する。
  - **凍結資産** — 実行メタデータ（`run_meta.json`）・集計結果CSV・章本文は
    確定後に凍結し、上書きしない。
- **これまでの主要所見**: 検討したシグナルの大部分について、市場オッズは公開情報を
  高い精度で織り込んでおり、執行可能な収益機会は検出されていません
  （＝市場効率性を支持するヌル結果が主体です）。
  **「未証明」は「反証済み」を意味しません。** 個別章の所見は連載記事および
  プレプリントで順次公開します。

## 記録対象と照合方法

| 資産 | 内容 | 記録状況 |
|---|---|---|
| `run_meta.json` | 実行時のパラメータ・乱数シード等 | 記録中（119件） |
| `chapter_text.md` | 章の本文（分析報告） | 記録中（20件） |
| `paper*_draft_v*.md` | 論文の内部草稿 | 記録中（15件） |
| 凍結CSV | 確定した集計結果 | 順次記録 |
| `*preregistration.md` | 計算実行前に固定した仮説・判定基準 | 遡及記録106件（下記2）／事前性の記録は CH23 以降 |

原本が公開された時点で（プレプリント公開時等）、第三者は次の1行で照合できます:

```bash
sha256sum <入手した原本ファイル>
# 出力が HASHES.md の該当行と一致すれば、そのファイルは
# 当該行を含むコミットの時点で確定していたことが検証できます
```

事前登録の記録は 2026-07-15 に、既存の事前登録文書106件の遡及バックフィルとして
始まりました（`note` 欄に `retroactive backfill` と明記。事前性の証拠ではありません。
下記「既知の限界」2参照）。事前性を主張できるのは `note` 欄に
`prereg (recorded before compute)` を含む行のみで、CH23 以降に記録が始まります。
第三者は `grep 'recorded before compute' HASHES.md` で、事前性が主張されている行だけを
抽出できます。その行を含むコミットが、同じ分析の `run_meta.json` を含むコミットより
前にあることを確認できます。

## 既知の限界（本レジストリの射程）

第三者が本レジストリの主張を正しく値引きできるよう、既知の限界を明記します。

1. **2026-07-13 より前の記録には外部証跡がありません。**
   本プログラムは 2026-07-13 まで git 管理外で運用されており、それ以前の資産の
   確定時刻を裏づける外部証跡は存在しません（研究側の append-only 台帳と
   ファイル mtime による自己申告のみ）。初回記録は一括の遡及記録であり、
   **各章の本来の確定日を証明するものではありません。** 証明されるのは
   「これらのファイルが記録時点でこの内容だった」ことのみです。
2. **事前登録文書の「事前性」は、2026-07-15 以前の章については立証されていません。**
   2026-07-15 に既存の事前登録文書106件を遡及的に記録しましたが、これらは
   計算実行後に記録されたものであり、`note` 欄に `retroactive backfill` と明示しています。
   証明されるのは「これらの文書が記録時点でこの内容だった」ことのみで、
   「結果を見る前に固定されていた」ことではありません。
   事前性を立証する記録（`prereg (recorded before compute)`）は CH23 以降に開始します。
   第三者は `grep 'recorded before compute' HASHES.md` で、事前性が主張されている行だけを
   抽出できます。
3. **本プログラムの全22章は 2026-06-15 〜 2026-07-15 の約1ヶ月間に実施されました。**
   したがって本レジストリの開設時点で、前向きに蓄積された検証データは存在しません。
   前向き検証はこれから始まります。
4. **blind は holdout としての効力を失っています。**
   blind 区間は研究期間中に複数回測定されており、厳密な意味での holdout では
   ありません。本プログラムの前向き検証面は lockbox のみです。
5. **lockbox は「一度きり開封する箱」ではなく開区間です。**
   2026-06-30 に、事前登録済みの単一仮説について1回開封しました（判定は FAIL）。
   その時点で消費された最終日は 2026-06-21 です。2026-06-22 以降は未参照であり、
   週次で伸びていきます。開封のたびに消費済み日付を前進させ、本 README に反映します。
6. **lockbox 抽出には第8〜12レースの絞り込みが適用されていません。**
   このため lockbox は train/validation/blind と母集団が異なり、
   split 間での集計値の直接比較はできません。
7. **過去に3つの互換性のない分割定義が並存していました。**
   初期の探索フェーズで用いた定義と現行の定義は日付体系が異なり、同じ区間を
   異なる名前で呼んでいた期間があります。現行の正規定義は上表のとおりです。
8. **本プログラムは大規模言語モデルによる自動パイプラインで運用されています。**
   判定基準の凍結・分割境界の裁定・lockbox の開封・本番反映は人間が行いますが、
   分析の実装と実行は自動化されています。

## 運用ルール

- `HASHES.md` は **append-only** です。行の削除・書き換えは行いません。
  ファイルが更新された場合は `updated` 注記付きの行を**追加**します。
- ハッシュの追記とコミットは研究パイプラインが各サイクルで自動的に行います。
- **事前登録文書のハッシュは、当該分析の計算実行に先立ってコミットします。**
  これにより「結果を見る前に判定基準が固定されていた」ことがコミット順序で確認できます。
